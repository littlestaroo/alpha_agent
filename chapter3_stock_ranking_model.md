# 第三章股票排序模型说明

## 1. 问题建模

在完成新闻/舆情文本的情绪分析、数值化与股票级因子生成之后，第三章后续的核心任务可以进一步建模为：

> 在交易日 $t$，给定股票池 $\mathcal{S}_t=\{s_1,s_2,\dots,s_n\}$ 及其对应的舆情因子向量 $\mathbf{F}_{s,t}$，设计一个排序函数 $g(\mathbf{F}_{s,t})$，对股票进行横截面打分排序，并选出分数最高的 `top k` 只股票。

也就是说，第三章最终不只是“生成因子”，而是要从因子进一步走向“排序与选股”。

## 2. 当前可用的因子分组

目前已经形成的股票级因子可以分成三类：

### 2.1 直接型因子

- `net_sentiment_factor`
- `negative_shock_factor`
- `attention_factor`
- `composite_score`

这一组回答的是：**今天这只股票的舆情整体怎么样。**

### 2.2 状态型因子

- `ema5_sentiment_state`
- `negative_shock_carry`
- `state_composite_factor`

这一组回答的是：**最近几天累积下来，这只股票的舆情状态怎么样。**

### 2.3 事件状态因子

- `earnings_event_state`
- `operations_event_state`
- `market_buzz_event_state`
- `risk_event_state`

这一组回答的是：**最近几天，哪一类事件正在持续发酵。**

## 3. 排序模型设计思路

考虑到不同因子的量纲不同、正负方向不同，本文采用“横截面百分位标准化 + 加权综合”的方式构造排序分数。

### 3.1 因子方向对齐

对于正向因子，例如净情绪、关注度、状态综合分，值越大通常越好。

对于负向因子，例如负面冲击、负面冲击记忆、风险事件状态，值越大反而越差。

因此，先对所有因子做方向对齐：

\[
\tilde{x}_{s,t}^{(f)} = d_f \cdot x_{s,t}^{(f)}
\]

其中：

- $x_{s,t}^{(f)}$ 表示股票 $s$ 在日期 $t$ 的第 $f$ 个因子值；
- $d_f \in \{+1,-1\}$ 表示该因子的方向。

### 3.2 横截面百分位标准化

在同一个交易日内，对每个方向对齐后的因子做横截面百分位排名：

\[
q_{s,t}^{(f)} = \text{PctRank}\left(\tilde{x}_{s,t}^{(f)}\right)
\]

这样每个因子都会被映射到 `[0,1]` 区间：

- 越接近 `1`，说明该股票在该因子上越靠前；
- 越接近 `0`，说明该股票在该因子上越靠后。

这样做的优点是：

1. 不同因子可以直接比较；
2. 不依赖正态分布假设；
3. 更适合后续做横截面排序。

### 3.3 加权综合分

本文将三类因子进一步组合成一个排序分数：

\[
\text{Score}_{s,t} = \sum_f w_f \cdot q_{s,t}^{(f)}
\]

当前 baseline 排序模型采用如下权重：

#### 直接型因子

- `net_sentiment_factor`：`0.08`
- `negative_shock_factor`：`0.04`
- `attention_factor`：`0.08`
- `composite_score`：`0.08`

#### 状态型因子

- `ema5_sentiment_state`：`0.14`
- `negative_shock_carry`：`0.09`
- `state_composite_factor`：`0.14`

#### 事件状态因子

- `earnings_event_state`：`0.05`
- `operations_event_state`：`0.17`
- `market_buzz_event_state`：`0.05`
- `risk_event_state`：`0.08`

可以看到：

- 直接型因子的总权重约为 `0.28`
- 状态型因子的总权重约为 `0.37`
- 事件状态因子的总权重约为 `0.35`

这种设计体现了当前第三章的核心判断：

> 单纯使用当天舆情值不够稳，因此在排序模型中，状态型因子和事件状态因子的权重高于直接型因子。

其中，`operations_event_state` 权重设置得更高，是因为当前小样本检验里它表现最强，说明经营事件持续发酵对后续相对强弱的解释力更突出。

## 4. 选股规则

在得到每只股票的排序分数之后，按日期做降序排列：

\[
s_{(1),t}, s_{(2),t}, \dots, s_{(n),t}
\]

然后直接选取得分最高的前 `k` 只股票，作为当日的候选股票集合：

\[
\mathcal{P}_t^{(k)} = \{ s_{(1),t}, s_{(2),t}, \dots, s_{(k),t} \}
\]

这就是本文的 `top k` 选股基线模型。

## 5. 当前实现状态

这一排序模型已经实现成脚本：

`analysis/build_stock_rankings.py`

其输入为日频因子表，输出为：

1. 完整横截面排序结果；
2. 每日 `top k` 股票名单；
3. 排序摘要 JSON。

需要特别说明的是：在大样本实验里，如果新闻是分批调用 LLM 的，那么**不能直接把不同批次的 `daily_profiles` 简单拼接**，因为同一个“日期-股票”可能会被拆到不同批次里，导致重复聚合。正确做法是：

1. 先合并不同批次的 `article_mentions`；
2. 再基于合并后的文章级结果重新全局构建 `daily_profiles`；
3. 最后再进行横截面排序和 `top k` 选择。

这一步当前也已经实现成辅助脚本：

`analysis/rebuild_profiles_from_mentions.py`

## 6. 当前 demo 结果

目前已经完成 `20` 只股票、`2023Q4` 候选新闻前 `5` 个批次的真实 API 处理。将五个批次的 `article_mentions` 合并后，再全局重建 `daily_profiles`，当前得到：

- `1,281` 条文章级结构化结果；
- `207` 条股票-日期级因子记录；
- `12` 个交易日可用于横截面排序；
- 平均每日横截面股票数约 `17.25`。

在这版全局重建结果上，已经可以直接输出每日 `top5` 股票名单。例如：

- `2023-10-10`：中国移动、比亚迪、爱尔眼科、海康威视、长江电力；
- `2023-10-11`：中国移动、长江电力、工商银行、三一重工、爱尔眼科；
- `2023-10-12`：工商银行、爱尔眼科、中国移动、比亚迪、美的集团。

这里的结果已经能够说明：

- 已有因子可以进一步组织成横截面排序模型；
- 模型可以直接输出每日 `top k` 股票；
- 多批次 LLM 结果已经可以汇总成统一的全局选股面板；
- 第三章后续已经可以从“因子生成”进入“排序选股”阶段。

## 6.1 当前排序效果的探索性验证

为了避免只停留在“能排序”，本文进一步将排序结果与后续收益对齐，做了一个轻量的 `top k` 选股效果验证。当前 `5` 个批次、`12` 个交易日样本下的结果为：

- 未来 `1D`：`top5` 平均超额收益约 `-0.0739%`，相对全市场命中率约 `50.0%`；
- 未来 `3D`：`top5` 平均超额收益约 `-0.0330%`，相对全市场命中率约 `58.3%`；
- 未来 `5D`：`top5` 平均超额收益约 `-0.3197%`，相对全市场命中率约 `33.3%`。

这说明当前排序 baseline 已经具备“从因子到 `top k` 股票选择”的完整形式，但排序效果仍处于探索阶段，后续仍需要：

1. 扩展更多股票与更长时间窗口；
2. 继续优化直接型、状态型和事件状态型因子的权重；
3. 在事件多样性筛选后，再比较不同因子设计的排序表现。

## 6.2 事件多样性筛选的对照结果

按照第三章目前的大样本路线，候选新闻并不是直接从 `top10` 压到 `top2`，而是先让 LLM 抽取 `event_type`，再对同一天同一只股票优先保留不同事件类型的新闻。当前在前 `5` 个批次上做 `diverse top3` 后：

- 文章级结果由 `1,281` 条压缩到 `546` 条；
- 但股票-日期级因子记录仍保持 `207` 条，说明事件筛选主要压缩了同质新闻，而没有破坏横截面覆盖。

进一步做 `top5` 排序验证后发现：

- 未来 `1D`：平均超额收益由 `-0.0739%` 改善到 `+0.0439%`；
- 未来 `3D`：平均超额收益由 `-0.0330%` 改善到 `+0.0009%`；
- 未来 `5D`：平均超额收益由 `-0.3197%` 变为 `-0.3856%`，中期仍然偏弱。

这说明“按事件类型二次筛选”在短期上有一定改善作用，至少能够减少同日同股的重复信息噪声；但从当前样本看，它还不足以单独解决中期排序效果偏弱的问题，后续仍需要结合更长时间窗口和更大股票池继续验证。

## 7. 论文中如何表述

在论文中，可以这样概括这个模型：

> 本文将股票舆情因子挖掘问题进一步建模为横截面排序问题。具体而言，在交易日内，首先对各股票的直接型因子、状态型因子和事件状态因子进行方向对齐与百分位标准化；随后通过加权求和构造综合排序分数；最后按分数从高到低排序，并选取得分最高的前 `k` 只股票，形成基于舆情另类数据的 `top k` 股票选择结果。

## 8. 当前最合适的定位

需要注意的是，当前这套模型的定位仍然是：

- **排序 baseline**
- **第三章方法闭环的重要一步**
- **为后续大样本验证与第四章智能体选股展示做准备**

因此，现阶段最重要的并不是强调它已经是最终交易模型，而是说明：

1. 已完成从新闻到因子的链路；
2. 已完成从因子到排序模型的过渡；
3. 后续将通过更大样本继续验证不同因子设计和排序结果的稳定性。
