# 第三章 Baseline 可视化与优化分析

## 1. 当前可视化结果说明

本次分析基于 `2023-11`、`宁德时代 + 贵州茅台`、按日每股 `top3` 新闻样本所生成的正式 baseline 结果展开。对应可视化文件如下：

1. `output/chapter3_visuals/chapter3_state_comparison.svg`
2. `output/chapter3_visuals/chapter3_event_decomposition.svg`
3. `output/chapter3_visuals/chapter3_enhanced_factor_demo.csv`

其中：

- `chapter3_state_comparison.svg` 主要比较“直接使用当天净情绪因子”和“引入时序累计后的状态值”之间的差异；
- `chapter3_event_decomposition.svg` 主要展示不同股票的事件结构差异，以及不同事件类型对应的平均情绪差异；
- `chapter3_enhanced_factor_demo.csv` 则演示了如何在当前 baseline 因子基础上进一步构造 `ema3_sentiment_state`、`ema5_sentiment_state`、`negative_shock_carry` 和 `state_composite_demo` 等增强型时序因子。

## 2. 当前直接方法的优点

当前方法最大的优点是简单、可解释、链路完整。它已经完成了：

`新闻文本 -> LLM 单条新闻情绪/事件分析 -> 股票日频聚合 -> 日频因子输出`

从实验角度看，单条新闻的舆情值直接由 LLM 给出，这一步本身是比较合理的，因为：

1. 新闻文本属于非结构化数据，LLM 在事件识别、情绪方向判断和摘要提取上明显优于纯关键词法；
2. 对单条新闻来说，LLM 更适合作为“文本理解器”，能够同时输出情绪、事件、风险和关键词；
3. 因此，单条新闻层面的结构化，当前继续沿用 LLM 是合适的。

也就是说，真正需要优化的重点，不是“单条新闻是否还要继续用 LLM”，而是“多条新闻聚合以后，应该如何把这些结构化结果变成更有决策价值的时序因子”。

## 3. 当前直接方法的不足

### 3.1 只看当天值，噪声偏大

如果直接把 `net_sentiment_factor` 当成决策信号使用，那么模型对当天新闻的敏感度会很高，但稳定性偏弱。

从当前增强演示结果看：

1. 宁德时代的当日净情绪因子标准差约为 `0.2507`，而 3 日 EMA 状态值标准差降到 `0.1398`，进一步构造的状态型综合值标准差降到 `0.1085`；
2. 贵州茅台的当日净情绪因子标准差约为 `0.3012`，3 日 EMA 状态值标准差降到 `0.2116`，状态型综合值标准差降到 `0.1702`。

这说明，同样的数据，如果只看当天值，波动会更大；一旦加入时间累计和负面冲击记忆，信号会明显更平滑。

### 3.2 只看总舆情值，会把不同事件混在一起

当前 baseline 的一个重要发现是：相同的总情绪值，背后可能来自完全不同的事件结构。

例如：

1. 贵州茅台的正向舆情，很多来自特别分红、提价、渠道改善和股东回报；
2. 宁德时代的波动，则更容易出现在市场竞争、供应链合作、行业调整和技术路线讨论中。

如果只看一个总的 `net_sentiment_factor`，这些差异会被压缩掉，难以支持更细的决策判断。

### 3.3 负面新闻的影响通常具有持续性

当前直接方法里，`negative_shock_factor` 是当天的截面量，但现实中负面事件往往会持续影响数日，甚至数周。

例如：

1. 竞争压力；
2. 价格战；
3. 监管处罚；
4. 业绩不及预期；
5. 核心客户变动。

如果这些负面事件只在当天记一次，后续影响会被低估。因此，负面冲击更适合设计成“带衰减记忆”的时序状态，而不是只做单日值。

### 3.4 当前还是“新闻驱动画像”，还不是“决策驱动信号”

当前 baseline 已经能做出“新闻解释 + 股票画像”，但还没有真正变成适合决策的信号体系。

原因在于：

1. 还没有引入时间状态记忆；
2. 还没有引入事件持续性和事件新颖性；
3. 还没有与价格、成交量、收益率等市场变量联动；
4. 还没有做因子有效性检验。

因此，当前更像“可解释的舆情画像”，而不是“可直接拿来交易或配置的因子系统”。

## 4. 更合理的优化方向

## 4.1 单条新闻继续用 LLM，多条新闻交给时序因子层

建议保留现在的模式：

1. 单条新闻：继续使用 LLM 做事件和情绪抽取；
2. 股票日频：交给规则聚合或统计模型做时序建模。

这是因为：

1. LLM 擅长理解文本；
2. 但多日累计、时序衰减、风险记忆和状态平滑，更适合用显式因子或时间序列模型处理。

换句话说：

**LLM 负责“看懂每条新闻”，时序模型负责“决定这些新闻应该怎样持续影响股票”。**

## 4.2 不建议只用当天舆情值，建议分三层使用

### 第一层：直接信号层

这一层保留当天值，适合短期响应型场景。

建议使用：

1. `net_sentiment_factor`
2. `negative_shock_factor`
3. `attention_factor`

适合回答：

- 今天这只股票舆情是偏正面还是偏负面？
- 今天是不是出现了明显风险冲击？
- 今天市场关注度有没有明显抬升？

### 第二层：状态信号层

这一层用于刻画情绪状态，不直接看当天，而看一段时间的累计结果。

建议构造：

1. `ema3_sentiment_state`
2. `ema5_sentiment_state`
3. `negative_shock_carry`
4. `state_composite_demo`

示意公式如下：

$$
State_t = \alpha \cdot NetSent_t + (1-\alpha)\cdot State_{t-1}
$$

$$
NegCarry_t = \beta \cdot NegCarry_{t-1} + NegShock_t
$$

其中：

1. `State_t` 表示情绪状态值；
2. `NegCarry_t` 表示负面冲击的递延影响；
3. `\alpha` 可以较大，使状态对最近几天更敏感；
4. `\beta` 可以略小，使负面冲击保留更长记忆。

这层更适合用于：

- 短中期走势判断；
- 连续多日情绪改善或恶化识别；
- 避免单日噪声干扰。

### 第三层：事件状态层

这一层不再只看总情绪，而是把事件拆开做。

建议按事件类型分别构造：

1. `earnings_state`
2. `operation_state`
3. `financing_state`
4. `market_buzz_state`
5. `product_state`

例如：

$$
EventState_{k,t} = \lambda \cdot EventState_{k,t-1} + CurrentEventScore_{k,t}
$$

其中 `k` 表示事件类型。

这样做的意义在于：

1. 业绩利好和市场补跌不能简单合并；
2. 回购、分红、渠道改善、价格战等信息对决策意义不同；
3. 事件拆分后，能更容易形成“解释性更强”的策略特征。

## 4.3 应该进一步提取“事件新颖性”和“事件持续性”

除了事件强度本身，还建议增加两个维度：

1. **事件新颖性**
   - 当前事件与过去 5 日或 10 日均值相比是否突然放大；
   - 如果是，则说明市场进入了新的叙事阶段。

2. **事件持续性**
   - 某类事件是否连续多日出现；
   - 如果某类负面事件连续出现，其决策意义通常高于单次出现。

这两个特征能帮助模型回答：

- 这是一次偶发新闻，还是新的趋势开始？
- 这是短期噪声，还是市场正在形成稳定预期？

## 4.4 应该进一步引入来源质量和重复新闻控制

当前 baseline 虽然已经做了候选筛选，但还可以继续增强：

1. 对新闻来源赋权；
2. 对高度重复的报道进行去冗余；
3. 区分“公司直接相关新闻”和“行业/板块跟风报道”。

因为在真实舆情环境下：

1. 证券研报摘要、
2. 宏观盘面点评、
3. 自媒体二次转载、
4. 公司公告解读

它们对决策的影响能力并不一样。

## 5. 如何更好地提升其对决策的影响能力

从“能看”到“能决策”，我建议按下面这条路线走：

### 路线 1：把当前 baseline 变成双层信号

先不大改模型，直接在当前结果上增强：

1. 保留当日直接因子；
2. 增加 3 日、5 日 EMA 状态因子；
3. 增加负面冲击记忆因子；
4. 用状态因子替代单日因子做后续检验。

这是最省时间、也最适合你当前论文节奏的改法。

### 路线 2：从总情绪值升级到事件因子

在当前文章级结果里，已经有 `event_type`，所以完全可以继续往下做：

1. 业绩事件因子；
2. 经营事件因子；
3. 投融资事件因子；
4. 市场舆情事件因子。

然后在股票-日期层面分别累计，而不是全部揉成一个总分。

这样你第三章就会从“舆情情绪因子”进一步升级到“事件-情绪联合因子”。

### 路线 3：与市场变量结合，形成真正的决策信号

下一步如果要提升决策影响力，最关键的是把这些舆情因子和市场变量结合：

1. 收益率；
2. 波动率；
3. 成交量；
4. 换手率；
5. 行业相对强弱；
6. 重大公告日期。

这样才能真正判断：

1. 哪些舆情信号能领先价格；
2. 哪些信号只是价格的同步反应；
3. 哪些信号在不同市场环境下有效性更强。

## 6. 当前最推荐的下一步

结合你现在的论文进度，我最推荐的顺序是：

1. 保留 LLM 作为单条新闻结构化工具；
2. 在日频层面新增 `ema3_sentiment_state`、`ema5_sentiment_state` 和 `negative_shock_carry`；
3. 再从现有 `event_type` 出发构造 2-4 个事件状态因子；
4. 最后再做轻量的因子检验和案例分析。

也就是说：

**短期最值得做的，不是推翻当前 baseline，而是在当前 baseline 上增加“时序累计层”和“事件状态层”。**

## 7. 一句话结论

如果只问“现在这批数据怎么用更好”，我的结论是：

**单条新闻的舆情值继续交给 LLM 做是合理的；但股票层面的决策信号不建议只用当天值，最好改成“当天直接值 + 时序累计状态 + 事件分解状态”三层并行。**
