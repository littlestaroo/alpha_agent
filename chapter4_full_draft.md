# 第四章 面向另类数据挖掘因子的智能体设计与实现

## 4.1 本章研究目标

第三章已经完成了从新闻文本到股票因子、再到横截面排序的模型设计与实验验证。但若仅停留在脚本级方法层面，仍难以支持真实研究流程中的自动化运行、结果追踪和后续扩展。因此，第四章进一步围绕第三章方法，设计并实现一套面向股票排序任务的多智能体工作流系统。

本章的目标主要包括：

1. 将第三章的方法流程系统化封装；
2. 通过多智能体分工协作实现自动运行；
3. 输出可直接用于分析和论文展示的结构化结果与系统报告；
4. 为后续扩展传统股票因子融合、多源数据接入和投研辅助系统奠定实现基础。

## 4.2 系统需求分析

从研究任务出发，本系统需要满足以下要求：

### 4.2.1 数据处理自动化

系统应能自动完成新闻读取、股票池读取、日期范围筛选和运行参数管理，减少研究者在数据准备阶段的重复操作。

### 4.2.2 文本分析模块化

系统应将单条新闻结构化分析独立封装，使得后续替换模型、提示词或数据源时不需要重写整个工作流。

### 4.2.3 因子构建与排序衔接

系统应能够在文章级结果生成后，自动聚合为股票级日频因子，并进一步调用排序模型输出每日 top k 股票。

### 4.2.4 评估与报告输出

系统应能自动将排序结果与价格数据对齐，输出收益评估、Markdown 报告和 JSON 摘要，使实验结果便于复现和展示。

### 4.2.5 可扩展性

系统应支持：

- 新增因子；
- 切换排序 preset；
- 扩展股票池；
- 接入传统市场因子；
- 替换新闻输入和价格输入。

## 4.3 多智能体架构设计

基于上述需求，本文将工作流拆分为五类 agent。

### 4.3.1 数据输入 Agent

数据输入 agent 负责统一管理系统输入，包括：

- 新闻文件路径；
- 股票池路径；
- 日期范围；
- 样本条数限制；
- 排序模型 preset；
- top k 参数。

这一 agent 的作用是为整个工作流建立一致的上下文。

### 4.3.2 舆情分析 Agent

舆情分析 agent 负责：

1. 对新闻进行股票候选匹配；
2. 调用 LLM 完成事件识别与情绪数值化；
3. 输出文章级结构化结果。

其输出字段包括：

- 股票对象；
- 情绪方向；
- 情绪强度；
- 相关度；
- 置信度；
- 事件重要度；
- 风险标记；
- 事件类型。

这一 agent 对应第三章中的“新闻级数值化”步骤。

### 4.3.3 因子构建 Agent

因子构建 agent 负责将文章级结构化结果进一步聚合为股票-日期级日频因子，输出包括：

- 直接型因子；
- 时序状态因子；
- 事件状态因子；
- 可靠性增强因子。

该 agent 是第三章因子模型在系统中的具体实现载体。

### 4.3.4 排序选股 Agent

排序选股 agent 负责：

- 因子方向对齐；
- 横截面百分位标准化；
- 加权排序；
- 可靠性收缩；
- 每日 top k 股票选择。

这一 agent 支持 `direct`、`state`、`event` 和 `optimized` 等多种 preset，使系统能够根据不同使用场景切换排序策略。

### 4.3.5 评估报告 Agent

评估报告 agent 负责：

1. 将排序结果与未来收益对齐；
2. 计算 top k 平均收益、超额收益、top-bottom spread 和命中率；
3. 生成 Markdown 报告和 JSON 摘要。

该 agent 使系统具备“从运行到展示”的闭环输出能力。

## 4.4 系统工作流设计

第四章系统整体工作流如下：

```text
数据输入 Agent
  -> 舆情分析 Agent
  -> 因子构建 Agent
  -> 排序选股 Agent
  -> 评估报告 Agent
```

在实现层面，其函数链路可以写为：

```text
run_baseline()
  -> build_daily_profiles()
  -> build_rankings()
  -> evaluate_topk()
  -> build_markdown_report()
```

该流程使得系统从新闻输入到 top k 股票选择和评估报告输出全部自动化完成。

## 4.7 系统运行实验

为验证第四章系统的可执行性，本文在双股票、单月样本上进行了 smoke run。运行配置如下：

- 新闻数据：`2023-11` 双股票按日单条样本；
- 股票池：宁德时代与贵州茅台；
- 排序模型：`optimized`；
- 选股规则：`top1`；
- 价格缓存：本地双股票复权价格数据。

实际运行结果表明，系统能够自动完成：

1. 文章级结构化分析；
2. 股票级因子构建；
3. 横截面排序；
4. top1 股票选择；
5. 未来收益评估；
6. Markdown 报告生成。

系统报告中给出的结果显示：

- `future 1D`：平均超额收益约 `0.1922%`；
- `future 3D`：平均超额收益约 `0.4217%`；
- `future 5D`：平均超额收益约 `0.6717%`。

需要说明的是，这一结果主要用于证明工作流端到端可运行，并不用于给出强统计结论。

## 4.8 系统特点分析

### 4.8.1 模块清晰

各个 agent 分工明确，数据输入、文本分析、因子构建、排序和评估相互独立，便于后续替换其中任一模块。

### 4.8.2 方法与系统一致

第四章系统并不是脱离第三章重新搭建，而是对第三章方法的直接工程实现，因此论文结构上具有较强一致性。

### 4.8.3 支持多策略切换

通过 preset 机制，系统可在不同排序策略间切换，这使得第四章系统不仅是“一个固定流程”，而是一个具有策略可配置能力的实验平台。

### 4.8.4 结果可追踪

系统在每个关键步骤保留 CSV、JSON 和 Markdown 结果，既方便研究者追踪过程，也有助于论文写作和答辩展示。

### 4.8.5 便于扩展

系统后续可进一步扩展为：

- 传统股票因子融合系统；
- 更长时间窗口大样本评估系统；
- 多源数据接入系统；
- 面向投研辅助的自动分析平台。

## 4.10 本章小结

本章围绕第三章提出的文本另类数据因子挖掘与排序方法，设计并实现了一套面向股票选择任务的多智能体工作流系统。该系统通过数据输入 agent、舆情分析 agent、因子构建 agent、排序选股 agent 和评估报告 agent 的协同工作，实现了从新闻输入到 top k 股票输出和收益评估报告生成的完整闭环。实验结果表明，该系统在工程上具备良好的可执行性和可扩展性，为本文后续研究以及实际投研辅助系统构建提供了实现基础。
