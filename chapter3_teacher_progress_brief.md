## 3. 当前直接方法的不足

老师提出的“当前这种直接方法有哪些不足、如何改进”，现在已经可以明确回答。

### 3.1 直接使用当天舆情值的问题

如果只直接使用当天的舆情值，例如当天净情绪或当天综合分，主要存在以下不足：

1. **噪声较大**  
   当天新闻条数少、题材偶然性强，容易导致单日值波动过大。

2. **缺乏时间持续性表达**  
   许多利好或利空不会只影响一天，直接日度值难以表达事件持续发酵的影响。

3. **不同事件被压成一个总分**  
   业绩变化、经营变化、风险事件、市场情绪在投资决策中的含义不同，但若只保留一个情绪总分，则解释力不足。

4. **对决策支持能力有限**  
   直接型因子更适合回答“今天发生了什么”，但不够适合回答“最近状态如何变化、哪类事件正在持续影响市场”。

## 4. 当前已经识别出的改进方向

基于上面的不足，第三章目前已经明确的改进方向是：

### 4.1 从“当天值”扩展为“状态值”

不只保留当天因子，而是引入时间累计与平滑：

- `ema3_sentiment_state`
- `ema5_sentiment_state`
- `negative_shock_carry`

这样可以更好地表达舆情的延续效应。

### 4.2 从“总情绪”扩展为“事件拆分”

不只做一个总情绪值，而是把信息拆成：

- 业绩事件
- 经营事件
- 投融资事件
- 风险事件
- 市场舆情事件

再进一步形成各自的事件状态因子。

### 4.3 从“新闻情绪值”扩展为“可解释的因子体系”

因此，当前已经形成的思路是：

**单条新闻继续由 LLM 做数值化；股票层面不再只直接使用当天值，而是形成“当天值 + 时序累计状态 + 事件状态”三层并行的因子体系。**

## 5. 当前还不能下强结论的原因

这里是向老师汇报时必须主动说明的部分。

虽然目前已经做了小样本初步检验，但由于当前只有：

- `2` 只股票
- `1` 个月时间窗口

所以目前还**不能严格判断这些因子的普遍有效性**。现在可以做的是：

- 在当前样本内比较不同因子的相对表现；
- 判断哪种因子设计方向更合理；
- 证明“从数值化到因子生成”的研究链路是可行的。

但还不能下结论说：

> 这些因子已经被大样本证明有效，或者已经具备稳定预测能力。

因此，当前检验结果更适合解释为：

**因子设计的探索性验证，而不是最终有效性证明。**

- 当前采用的是“每天每股先保留 `top10` 候选新闻”的策略，而不是一开始就压成 `top2`。

目前真实抽取结果为：

- 扫描原始中文新闻：`8,781,093` 条；

这样做的目的，是先保留更宽的新闻候选池，等 LLM 提取出 `event_type` 之后，再做一次“事件多样性筛选”，尽量保证同一天同一只股票保留下来的新闻不是同一类型。

目前这条大样本链路已经不只是 smoke test，而是已经完成前 `5` 个批次的真实运行：

- `batch_000` 到 `batch_004` 全部成功完成；

- 再全局重建后，形成 `207` 条股票-日期级因子记录，可用于 `12` 个交易日的横截面排序。

这说明当前工作已经从“候选新闻抽取”推进到“多股票、多日期、真实 API 因子生成”的实施阶段。

## 6. 现在最应该做的事情

如果继续推进第三章，当前最重要的工作不是重新改写单条新闻数值化，而是：

### 6.1 扩大验证样本

建议将当前的小样本实验扩展为：

- 时间上：从 1 个月扩展到 6-12 个月；
- 股票上：从 2 只扩展到 10-20 只以上。

这样才能更稳地检验：

- 直接型因子是否稳定；
- 时序累计因子是否优于当天值；
- 事件状态因子是否真正优于总情绪值。

### 6.2 比较三类因子的表现差异

下一步最核心的实验问题应该是：

1. 直接型因子是否有效？
2. 时序累计因子是否优于直接型因子？
3. 事件状态因子是否优于总情绪因子？

也就是说，第三章后续重点应该从“能不能生成因子”，转向“哪一类因子设计更优”。

### 6.3 将当前结果定位为“第三章 baseline + 改进方向探索”

当前最适合的论文写法是：

- 先把“LLM 直接数值化 + 基础因子生成”作为 baseline；
- 再提出改进方向：
  - 时序累计因子
  - 事件状态因子
- 最后将因子进一步组织成横截面排序模型，对股票进行排序并选出 `top k`，再用扩样本实验比较不同因子设计与排序结果的表现。

### 6.4 最终模型形式：基于因子的股票排序与 top-k 选择

从老师现在提出的目标看，第三章最终的模型形式可以明确为：

- 输入：股票在日期 `t` 上的舆情因子向量；
- 模型：对因子做方向对齐、横截面标准化和加权求和，形成排序分数；
- 输出：按分数对股票排序，并选出前 `k` 只股票。

也就是说，当前第三章后续不再只是研究“因子能不能生成”，而是要进一步完成：

**因子 -> 排序分数 -> top k 股票选择**

这一步目前也已经有了可运行的 baseline 排序脚本。当前在五个真实批次、`20` 只股票的实验上，已经可以对 `12` 个交易日进行横截面排序，平均每日约有 `17.25` 只股票参与排序，并能直接输出每日 `top5` 股票名单。

同时，我还补做了一个轻量的选股效果验证：把 `top5` 结果和后续价格对齐后，当前未来 `3D` 的相对全市场命中率约为 `58.3%`，而 `1D/5D` 表现仍然偏弱。这说明排序模型已经跑通，但当前结果更适合解释为“探索性 baseline”，还需要继续扩样本和调权重。
